# Track: Champions' Wall

Montreal-shaped track (Circuit Gilles Villeneuve / Île Notre-Dame as inspiration). The name riffs on the "Wall of Champions" final chicane while staying clear of trademarks.

## Completed
- Centerline generated by `scripts/gen-tracks.mjs` via `catmullRomLoop()` through 26 hand-placed control points. 260 sampled centerline points.
- **Scaled 2× from initial version** for room around distinctive corners. Track footprint ~5400×2800 world units. Image scale 4 (world = (img_x − 750) × 4, (img_y − 468.5) × 4) used to map reference image pixels directly to world coordinates.
- Control points traced from the reference brown centerline using exact image pixel positions, then converted via the formula above. The asphalt centerline now runs through the brown reference line in every section: Lance Stroll hairpin, 47 chicane, main-straight diagonal, descent through Privilege cluster, right hairpin around the 11/8-4 grandstands, west-bound run past PLATINE/Senna Club/Podium Club, Family Grandstand chicane, Casino-equivalent back-straight, 31 chicane, west-side climb.
- Min curvature radius 88 (above half-width 70). Zero self-intersections.
- Authored against a semi-transparent reference overlay of the real Montreal map embedded in the inspector (toggle `4`). The overlay is loaded from `public/inspect-overlays/champions-wall.png` — gitignored to avoid shipping copyrighted material.
- Schema v2; grass runoff (outside 90, inside 50); no patches yet.
- **Racing-line hints in T12→T13 approach.** The minimum-curvature solver clamped the line at +48 (inside edge) for indices 195–200, leaving AI no buffer to grass. Two hints in `racingLineOverrides.hints` (idx 195 → offset 0, idx 198 → offset 10) pull the approach back near the centerline; the apex itself (idx 199–200) still snaps to the inside, which is correct racing-line geometry. Indices are tied to control-point order — reordering control points will shift them.
- Track width 140; min curvature radius 172 (well above half-width 70).
- `MenuScene.TRACKS` entry; selectable from menu. `startIndex = 80` places the start/finish near the checker on the reference map.
- Catmull-Rom helper `catmullRomLoop(controls, samplesPerSegment)` and chained `chain()` builder added to `gen-tracks.mjs`. Catmull-Rom is the right tool for irregular real-world tracks where iterating arcs+lines for closure is painful.

## Architecture Decisions
- **Centripetal Catmull-Rom over uniform.** Uniform Catmull-Rom overshoots through unevenly-spaced control points and produces tighter-than-expected curvature at sharp angles. Centripetal (alpha = 0.5) avoids the cusps and keeps curvature radius proportional to point spacing.
- **Reference overlay lives in track JSON, but image lives gitignored.** Schema field `referenceOverlay` (image path + world transform) is part of TrackData so the inspector can render the right overlay per track. The image file itself is excluded from git via `public/inspect-overlays/`.

## Constraints (learned the hard way)
- **Centerline curvature radius must stay above the track half-width** or `offsetLoop()` produces a self-intersecting polygon and the asphalt fill fragments. With width 110 (half = 55), every centerline triple must have circumscribed radius > 55. Two practical rules: (1) keep per-control-point heading change ≲ 60°, (2) spread tight corners across 4–5 control points. The Lance Stroll loop here uses 5 points; min-radius ~91.
- **Centerline must not self-intersect (no figure-8).** First version had the hairpin approach and exit both running south of the loop at similar y, so the smoothed Catmull-Rom curves crossed near the main-straight entry. Fix: place the approach to the *west* of the hairpin loop (climb up the west side from the 31-chicane area) so approach and exit meet only at the loop apex. After the fix: zero segment-segment intersections.
- Two sanity checks worth running after every regen — both implemented as a one-liner in the `node -e` block in `gen-tracks.mjs` review:
  1. Min 3-point circumscribed radius across the centerline > half-width.
  2. Zero pairwise segment-segment intersections (excluding adjacent-segment shared endpoints).

## Open Questions
- Lance Stroll loop reads as a wide U-turn rather than a tight 180° — at the new scale a tight 180° is now feasible if we pack 6+ control points around the apex.
- Wall of Champions chicane, Senna chicane, Family Grandstand chicane, and 31 chicane are all currently smoothed-through. Adding character will likely need either: (a) closely-spaced control-point pairs flipping perpendicular sign at the apex, or (b) splicing a `chain()` arc segment into the centerline at the chicane positions (hybrid Catmull-Rom + chain).
- Per-segment runoff widths and gravel/asphalt patches not yet authored — would help differentiate corner exits and add escape zones.
